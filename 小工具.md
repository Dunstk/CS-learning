___在vim中插入___
- **按** `Esc`：确保你处于普通模式。
    
- **按下插入命令**：例如按 `i`。
    
- **开始输入内容**：你会看到 Vim 左下角显示 `-- INSERT --`，表示你已进入插入模式。
    
- **退出插入模式**：按 `Esc` 键即可返回普通模式

___在 Vim 中复制粘贴___
按 `Esc`，然后输入 `yy`
按 `p` 粘贴到下一行，或 `P` 粘贴到上一行
![[Pasted image 20251102104810.png]]
___退出vim___

|命令|作用说明|
|---|---|
|`:q`|退出 Vim（如果没有修改）|
|`:q!`|强制退出，不保存修改|
|`:wq`|保存并退出（write + quit）|
|`:x`|保存并退出（与 `:wq` 类似）|
|`ZZ`|保存并退出（在普通模式下按两次 `Z`）|
- **按** `Esc`：确保你处于普通模式（不是插入模式）。
    
- **输入命令**：
    
    - 输入 `:`（英文冒号），光标会跳到底部命令行。
        
    - 然后输入 `q`、`q!`、`wq` 或其他命令。
        
- **按回车** `Enter`：执行命令

**注意：**
在 Git 中，代码分为三个区域：
1. **工作区（Working Directory）**：你正在编辑的文件内容
2. **暂存区（Staging Area）**：准备提交的内容
3. **版本库（Repository）**：已经提交的历史记录
如果你在某个分支（如 `hot-fix2`）中修改了文件，但没有执行 `git add` 和 `git commit`，这些修改就仍然停留在工作区。当你切换到另一个分支（如 `hot-fix3`）时：
> **Git 会自动把这些未提交的修改带到新分支中。**

这就导致你看到两个分支都“应用”了这份修改，实际上它们只是共享了同一个工作区状态

### 5.2找回root密码（经典面试题）
第一步要快！（5秒之内）
![[Pasted image 20251108194832.png]]

2、进入编辑页面，使用键盘上的上下键把光标往下移动，找到以““Linux16”开头内容所在的行数”，在行的最后面输入：init=/bin/sh
![[Pasted image 20251108195138.png]]

3、接着，输入完成后，直接按快捷键：ctrl+x进入单用户模式

4、在光标闪烁位置输入：mount -o remount,rw /  （斜杠与rw有空格）完成后按回车键enter

5、在新的一行的最后面输入：passwd，完成后按回车键。输入密码，然后再次确认密码即可，密码修改成功后，会显示passwd······的样式，说明密码修改成功

6、在光标闪烁位置输入：touch/.autorelabel（注意：touch与/后面有一个空格），完成后按回车

7、继续输入：exec/sbin/init（注意：exec与/后面有一个空格），完成后按键盘的回车键，等待long long time，系统会自动重启

>[!bug]
>- 利用systemctl查看服务自启动状态时必须写全名（带上service）
>- 利用其基本语法时可以不带service

![[Pasted image 20251111095911.png]]

>[!pay attention]
>①利用systemctl start/stop 服务名 语法对某服务操作，只是临时起效。当重启系统后，还是回归以前对服务的设置（自启动/非自启动）
②希望设置某个服务自启动或永久关闭，需要利用
systemctl disable/enable
